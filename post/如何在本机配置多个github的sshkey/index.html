如何解决Key is already in use报错

当我们想要在一台机器上开发多个不同的github项目时，我们就需要配置多个不同的公钥。github站点限制了一个ssh公钥只能在一个仓库用一次，不然就会提示Key is already in use。当我们在本机使用多个ssh公钥时，又该如何管理呢？本文以我的一个项目在ubuntu18.04上的操作作为演示。

基础信息介绍

# ssh配置所在目录
/home/ubuntu/.ssh/config

# 在github为项目添加的ssh公钥
/home/ubuntu/Projects/github-ssh-key/golang-todo/id_rsa

# github项目所在目录，也就是git clone的那个目录
/home/ubuntu/Projects/golang-todo

# 项目的地址
git clone git@github.com:panxl6/golang-todo.git

如果我们不做任何配置，那么在git push的时候，会使用id_rsa这个默认的公钥。

配置ssh，指定公钥文件

修改此文件，添加配置：$HOME/.ssh/config

1
2
3
4
5

Host github-golangtodo
  HostName github.com
  User git
  IdentityFile /home/ubuntu/Projects/github-ssh-key/golang-todo/id_rsa
  IdentitiesOnly yes

添加远程仓库

1

git remote add github-golangtodo git@github-golangtodo:panxl6/golang-todo.git

提交到远程仓库

1

git push -v github-golangtodo master
github ssh
